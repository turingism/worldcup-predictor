> 比賽日跑預測器的一頁式操作指南。專案是什麼看主 [README](../README.zh-CN.md)。

### ▶️ 啟動
```bash
cd ~/worldcup-predictor && python3 app.py        # 开 http://127.0.0.1:8000
```
- 連接埠固定 **8000**（別用 5000，被 macOS AirPlay 佔用）。
- 頁面空白/卡住？ **`Cmd+Shift+R` 硬重新整理**（清前端快取）。
- 維護機上也可以用 `launchd` 常駐：
  ```bash
  launchctl print gui/$(id -u)/com.melvin.worldcup-predictor
  ```

### 🤖 開著網頁就自動發生（無需動手）
| 頻率 | 做什麼 |
|---|---|
| 每 60 秒 | 看板刷新即時比分；LIVE 卡的 in-play 勝平負隨比分跳動 |
| 每 3 分鐘 | 靜默拉 ESPN 完賽 → **有新完賽就自動重新訓練模型**、刷新看板、背景重算奪冠 90% 區間（不用手動點「刷新事實資料」）|
| 每 30 分鐘 | 背景抓一次 ESPN 盤口（DraftKings 1X2），累積開盤/閉盤線 → 餵市場/CLV 層 |

> 讓**看板 tab 留在前景**，這些自動拉取才會觸發。比賽日把頁面開著就行——比分、完賽、奪冠區間、盤口全自動往前滾。

### 🖱️ 按鈕 / Tab 速查
- **📋 賽事看板**（首頁）：🔴 正在比賽 / 🟡 即將開賽 / ✅ 已結束。每場「看預測」彈出比分矩陣（與該行同口徑），每行「🔍 解讀」「📒 復盤」一鍵直達對應 tab。**預測驗證**帳本（賽前預測開球前凍結 vs 實際、逐場評分）就在「已結束」區——開賽初期看 RPS/校準，別盯命中率。
- **⚽ 對陣分析**（足球經理人）：選兩隊 → 資深分析師報告（過程資料 → Dixon-Coles 比分矩陣+熱力圖 → 1X2 / 大小球 / 亞盤 / 正確比分 / 信心度）。已併入原「單場預測」。
- **🔍 機制解讀**：單場市場機制解釋卡（Shin 去水水位結構 + 模型 vs 市場分歧，強制掛「市場對、模型錯」CLV 先驗），純描述性認知。
- **📒 競彩復盤**：手動錄入競彩讓球線與自己的判斷，賽後手填 90 分鐘結算比分 → 單場 我/模型/市場 三方對帳，用於學習和校準記錄。
- **🌳 晉級樹狀圖**：2026 官方括號投影；改任意比分/淘汰賽結果做假設 → 括號 + 奪冠機率即時重算。此 tab 的 **🔄 刷新事實資料** 手動秒拉 ESPN 完場（有新的就重新訓練）。
- **🏆 奪冠機率 + 實力榜**：蒙地卡羅點估 + 貝氏 90% 區間（whisker 圖，「↻ 重新計算」手動刷新）+ 貝氏淨實力榜。
- **💹 市場對標**：模型 vs 閉盤線 + CLV；價值/Kelly 面板**預設誠實鎖定**。
- **🔮 玄學占卜**：確定性的文化/演算法彩蛋，明確不是預測，並與基線一起對照。

### 🔧 兩個開關（環境變數）
```bash
READONLY=1 python3 app.py        # 只读分享模式：禁所有写接口（对外分享用）
MARKET_UNLOCK=1 python3 app.py   # 强制打开价值/Kelly 面板（仅演示，大红警示，非建议）
```

### 🩹 疑難排解速查
- **看板卡在「載入…」** → 硬重新整理；還不行就重啟 `app.py`。
- **8000 連接埠斷線** → 如果啟用了 `launchd`，執行 `launchctl kickstart -k gui/$(id -u)/com.melvin.worldcup-predictor`。
- **市場 tab 慢/空** → 開賽初期正常（還沒有「已完賽且抓到盤口」的場次 → 「樣本不足」）。
- **刷新回報 403** → 你在 `READONLY=1` 模式（寫入介面被刻意停用）。
- **資料陳舊** → 點「🔄 刷新事實資料」，或重啟 `app.py`。
