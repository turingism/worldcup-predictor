> 比赛日跑预测器的一页纸操作指南。项目是什么看主 [README](../README.zh-CN.md)。

### ▶️ 启动
```bash
cd ~/worldcup-predictor && python3 app.py        # 开 http://127.0.0.1:8000
```
- 端口固定 **8000**（别用 5000，被 macOS AirPlay 占）。
- 页面空白/卡住？ **`Cmd+Shift+R` 硬刷新**（清前端缓存）。
- 维护机上也可以用 `launchd` 常驻：
  ```bash
  launchctl print gui/$(id -u)/com.melvin.worldcup-predictor
  ```

### 🤖 开着网页就自动发生（无需动手）
| 频率 | 干什么 |
|---|---|
| 每 60 秒 | 看板刷新实时比分；LIVE 卡的 in-play 胜平负随比分跳 |
| 每 3 分钟 | 静默拉 ESPN 完赛 → **有新完赛就自动重训模型**、刷新看板、后台重算夺冠 90% 区间（不用手动点「刷新事实数据」）|
| 每 30 分钟 | 后台抓一次 ESPN 盘口（DraftKings 1X2），积累开盘/闭盘线 → 喂市场/CLV 层 |

> 让**看板 tab 留在前台**,这些自动拉取才会触发。比赛日把页面开着就行——比分、完赛、夺冠区间、盘口全自动往前滚。

### 🖱️ 按钮 / Tab 速查
- **📋 赛事看板**（首屏）：🔴 正在比赛 / 🟡 即将开赛 / ✅ 已结束。每场「看预测」弹比分矩阵（与该行同口径），每行「🔍 解读」「📒 复盘」一键直达对应 tab。**预测验证**账本（赛前预测开球前冻结 vs 实际、逐场打分）就在「已结束」区——开赛初期看 RPS/校准，别盯命中率。
- **⚽ 对阵分析**（足球经理人）：选两队 → 资深分析师报告（过程数据 → Dixon-Coles 比分矩阵+热力图 → 1X2 / 大小球 / 亚盘 / 正确比分 / 置信度）。已并入原「单场预测」。
- **🔍 机制解读**：单场市场机制解释卡（Shin 去水水位结构 + 模型 vs 市场分歧，强制挂「市场对、模型错」CLV 先验），纯描述性认知。
- **📒 竞彩复盘**：手动录入竞彩让球线与自己的判断，赛后手填 90 分钟结算比分 → 单场 我/模型/市场 三方对账，用于学习和校准记录。
- **🌳 晋级树状图**：2026 官方括号投影；改任意比分/淘汰赛结果做假设 → 括号 + 夺冠概率实时重算。此 tab 的 **🔄 刷新事实数据** 手动秒拉 ESPN 完场（有新的就重训）。
- **🏆 夺冠概率 + 实力榜**：蒙特卡洛点估 + 贝叶斯 90% 区间（whisker 图，「↻ 重新计算」手动刷）+ 贝叶斯净实力榜。
- **💹 市场对标**：模型 vs 闭盘线 + CLV；价值/Kelly 面板 **默认诚实锁定**。
- **🔮 玄学占卜**：确定性的文化/算法彩蛋，明确不是预测，并与基线一起对照。

### 🔧 两个开关（环境变量）
```bash
READONLY=1 python3 app.py        # 只读分享模式：禁所有写接口（对外分享用）
MARKET_UNLOCK=1 python3 app.py   # 强制打开价值/Kelly 面板（仅演示，大红警示，非建议）
```

### 🩹 排错速查
- **看板卡在"加载…"** → 硬刷新；还不行就重启 `app.py`。
- **8000 端口掉线** → 如果启用了 `launchd`，运行 `launchctl kickstart -k gui/$(id -u)/com.melvin.worldcup-predictor`。
- **市场 tab 慢/空** → 开赛初期正常（还没有"已完赛且抓到盘口"的场次 → "样本不足"）。
- **刷新报 403** → 你在 `READONLY=1` 模式（写接口被刻意禁用）。
- **数据陈旧** → 点「🔄 刷新事实数据」，或重启 `app.py`。
